# HITL Review Guide (GH-25)

## Overview

This guide describes the Human-in-the-Loop (HITL) review process for the
reprorusted-python-cli corpus. Senior Rust engineers review a 5% sample
to identify anti-patterns that automated tests cannot catch.

## Review Checklist

Each example should be evaluated against the following criteria:

### 游댮 no_unsafe
**Severity:** CRITICAL

No unnecessary `unsafe` blocks

### 游 minimal_cloning
**Severity:** HIGH

Minimal cloning (prefer borrowing)

### 游 idiomatic_error_handling
**Severity:** HIGH

Idiomatic error handling (Result/Option, no unwrap in lib code)

### 游리 iterator_usage
**Severity:** MEDIUM

Appropriate use of iterators (no explicit loops where iter works)

### 游리 no_pythonisms
**Severity:** MEDIUM

No Python-isms (Arc<Mutex<>> overuse, String everywhere)

### 游리 proper_lifetimes
**Severity:** MEDIUM

Proper lifetime annotations where needed

### 游릭 no_magic_numbers
**Severity:** LOW

No magic numbers (use constants)

### 游릭 documentation
**Severity:** LOW

Public APIs have doc comments

## Review Process

1. **Generate Sample**: `make corpus-hitl-sample`
2. **Review Each Example**: Check against the checklist above
3. **Document Findings**: Update `data/hitl_reviews/YYYY-QN-review.json`
4. **Report Summary**: `make corpus-hitl-report`

## Status Values

- `pending`: Not yet reviewed
- `approved`: Passes all critical/high checks
- `needs_work`: Has issues that need fixing
- `rejected`: Has critical anti-patterns

## Quarterly Schedule

- Q1: January-March
- Q2: April-June
- Q3: July-September
- Q4: October-December
